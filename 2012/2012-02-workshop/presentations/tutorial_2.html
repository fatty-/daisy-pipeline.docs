MISSION: To the new Pipeliners.

We have been informed from the Daisy Headquarters that some of the current developers from the pipeline 2 have deserted. These rascal dissidents have left the new Daisy Pipeline 2 C# client halfway through. Your mission is finishing their work within an estimated time of 2 hours. If you fail Dr.Evil will kill the attached super cute kittens.

Your weapons for this task are:

  • Daisy Pipeline WS API http://code.google.com/p/daisy-pipeline/wiki/WebServiceAPI
  • A C# project with some of the key parts missing.
  • Your developing skills.

Your check points are:

  1. Familiarise yourselves with the code.
  2. Show the list of available scripts on the screen.
  3. Show the details for a given script id.
  4. Submit a job.
  5. Show the job status.
  6. Bonus stage: delete a job.
    7. GOOD LUCK!

Remote Mode:

  1. You will have to modify Authentication.cs to build the proper authenticated URI. Follow the api to get how to do it.
  2. Keep in mind that in remote mode:
    • You have to provide the data file. So add the parameter --job-data=PATH_TO_ZIP
    • The inputs must be relative to the zip file (e.g. inside the zip file you have "/dir/file.xml" then your input have to be something like "./dir/file.xml")
